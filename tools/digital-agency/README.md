# AI Digital Agency

Система локальных AI-агентов для замены команды специалистов одним разработчиком.

## Назначение

Помогает разработчику:

1. Общаться с клиентом через агента (аккаунт-менеджер)
2. Находить эффективные связки «Охват → Слежение → Продажа»
3. Разрабатывать по спецификации из `/wiki/`

**Ключевой принцип:** стратегия строится на поиске связок — цепочек каналов, где каждый шаг конвертирует аудиторию (VK Клипы → подписка на Telegram → консультация с менеджером).

Агенты работают **локально**, не на продакшене.

## Быстрый старт

Когда клиент приходит с проектом:

```
Запусти этап 01: Сбор информации о клиенте
```

Клод автоматически:

1. Загрузит роль из `agency_pipeline_first_messages.md`
2. Задаст вопросы из `templates/client_brief_template.md`
3. Создаст артефакт в `/wiki/client_brief.md`

## Принципы работы

### Связки «Охват → Слежение → Продажа»

Стратегия = набор связок. Каждая связка — это цепочка:

```
Охват (алгоритмы показывают контент)
  → Слежение (аудитория подписывается через лидмагнит)
    → Продажа (подписчик становится клиентом через оффер)
```

Цикл: тестируем связку → находим слабое звено → усиливаем → масштабируем.

### Артефактная коммуникация

Агенты не "разговаривают" — создают документы. Каждый артефакт:

- Завершён и самодостаточен
- Читаем следующим агентом без интерпретации
- Сохраняется в `/wiki/` для MCP серверов

### Измеримость

Все решения через показатели:

- Возврат инвестиций
- Стоимость привлечения
- Жизненная ценность клиента
- Конверсия
- Удержание

### Цикл оптимизации

После этапа 05 → возврат к этапу 02 для корректировки стратегии.
