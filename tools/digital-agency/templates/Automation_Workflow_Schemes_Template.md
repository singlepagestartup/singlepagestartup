### 3. Брейншторм: обработка лидов от холодного к горячему

#### 3.1. Этапы прогрева лида

| Этап         | Статус лида     | Характеристики                        | Критерии перехода                                        | Источник данных                                                          |

| Холодный лид | Новый           | Только заявка, нет взаимодействий     | Создан в CRM через форму                                 | CRM_Integration_Map.md → Раздел 3 → №1                                   |
| Теплый лид   | В работе        | Есть открытия писем, клики, просмотры | Открыл письмо или кликнул по ссылке                      | Email_Tech_Setup.md → Раздел 3, Analytics_Tracking_Spec.md → Раздел 2    |
| Горячий лид  | Готов к продаже | Множественные взаимодействия, запросы | Кликнул по CTA, запросил консультацию, добавил в корзину | Analytics_Tracking_Spec.md → Раздел 2, CRM_Integration_Map.md → Раздел 3 |

#### 3.2. Стратегии обработки лидов

| Стратегия          | Цель                     | Методы                                               | Система               | Влияние на метрики        |
| ------------------ | ------------------------ | ---------------------------------------------------- | --------------------- | ------------------------- |
| Образование        | Повысить осведомленность | Полезный контент, кейсы, отзывы                      | ESP (Email)           | ↑ Retention, ↑ LTV        |
| Персонализация     | Увеличить вовлечение     | Сегментация по поведению, персонализированные офферы | ESP + CRM             | ↑ Conversion Rate, ↓ CAC  |
| Триггерные события | Ускорить конверсию       | Реакция на действия (клики, просмотры, корзина)      | ESP + CRM + Analytics | ↑ Conversion Rate, ↑ ROMI |
| Реактивация        | Вернуть неактивных       | Повторные контакты, специальные предложения          | ESP + CRM             | ↑ Retention, ↑ LTV        |

### 4. Автоматизационные сценарии (воронки продаж)

#### 4.1. Welcome Flow (Приветственная воронка)

**Цель:** Приветствие нового лида, установление первого контакта, повышение узнаваемости бренда.

| Этап | Триггер          | Действие                            | Система | Задержка | Условие ветвления          | Цель                  | Источник данных                                |
| ---- | ---------------- | ----------------------------------- | ------- | -------- | -------------------------- | --------------------- | ---------------------------------------------- |
| 1    | lead_created     | Отправка Welcome Email              | ESP     | 0 минут  | -                          | Приветствие           | Email_Tech_Setup.md → Раздел 3 → Welcome Email |
| 2    | email_opened     | Пометка "активный лид" в CRM        | CRM     | 0 минут  | Если открыл → теплый лид   | Сегментация           | CRM_Integration_Map.md → Раздел 3              |
| 3    | email_not_opened | Повторная отправка через 3 дня      | ESP     | 3 дня    | Если не открыл → повтор    | Достижение вовлечения | Email_Tech_Setup.md → Раздел 3 → Follow-up     |
| 4    | email_clicked    | Создание задачи в CRM для менеджера | CRM     | 0 минут  | Если кликнул → горячий лид | Продажа               | CRM_Integration_Map.md → Раздел 3              |

**Требуется ручная работа:** Да (человек должен создать воронку Welcome Flow в ESP, настроить триггеры, связать с CRM, протестировать отправку).

**Последовательность действий для человека:**

1. Создать воронку Welcome Flow в ESP (Mailchimp/SendPulse/UniSender/HubSpot/Brevo).
2. Настроить триггер "lead_created" из CRM (через webhook или API согласно CRM_Integration_Map.md → Раздел 3).
3. Добавить шаблон Welcome Email из Email_Tech_Setup.md → Раздел 3.
4. Настроить условия ветвления (открыл/не открыл, кликнул/не кликнул).
5. Настроить задержки (0 минут для первого письма, 3 дня для повторного).
6. Настроить интеграцию с CRM для создания задач при клике по ссылке (согласно CRM_Integration_Map.md → Раздел 3).
7. Протестировать воронку: создать тестового лида в CRM, проверить отправку письма, проверить создание задачи в CRM.
8. Предоставить результаты тестирования ИИ-агенту для фиксации в разделе 7.

#### 4.2. Nurturing Flow (Воронка прогрева лида)

**Цель:** Образование клиента, поддержание интереса, перевод холодного лида в теплый.

| Этап | Триггер             | Действие                       | Система | Задержка | Условие ветвления                 | Цель                  | Источник данных                             |
| ---- | ------------------- | ------------------------------ | ------- | -------- | --------------------------------- | --------------------- | ------------------------------------------- |
| 1    | form_submit         | Email: продуктовая информация  | ESP     | 1 день   | -                                 | Образование клиента   | Email_Tech_Setup.md → Раздел 3 → Follow-up  |
| 2    | email_not_opened    | Повторная отправка через 2 дня | ESP     | 2 дня    | Если не открыл → повтор           | Достижение вовлечения | Email_Tech_Setup.md → Раздел 3 → Follow-up  |
| 3    | email_opened        | Пометка "активный лид" в CRM   | CRM     | 0 минут  | Если открыл → теплый лид          | Сегментация           | CRM_Integration_Map.md → Раздел 3           |
| 4    | email_clicked       | Переход к кейсам/отзывам       | ESP     | 0 минут  | Если кликнул → горячий лид        | Прогрев               | Email_Tech_Setup.md → Раздел 3 → Newsletter |
| 5    | email_clicked + CTA | Создание задачи в CRM          | CRM     | 0 минут  | Если кликнул по CTA → горячий лид | Продажа               | CRM_Integration_Map.md → Раздел 3           |

**Требуется ручная работа:** Да (человек должен создать воронку Nurturing Flow в ESP, настроить триггеры, условия ветвления, связать с CRM, протестировать).

**Последовательность действий для человека:**

1. Создать воронку Nurturing Flow в ESP.
2. Настроить триггер "form_submit" из Analytics_Tracking_Spec.md → Раздел 2 (через webhook или API).
3. Добавить шаблоны писем из Email_Tech_Setup.md → Раздел 3 (Follow-up, Newsletter).
4. Настроить условия ветвления (открыл/не открыл, кликнул/не кликнул, кликнул по CTA).
5. Настроить задержки (1 день для первого письма, 2 дня для повторного).
6. Настроить интеграцию с CRM для обновления статуса лида (согласно CRM_Integration_Map.md → Раздел 3).
7. Протестировать воронку: отправить тестовую заявку через форму, проверить отправку писем, проверить обновление статуса в CRM.
8. Предоставить результаты тестирования ИИ-агенту для фиксации в разделе 7.

#### 4.3. Abandoned Cart Flow (Воронка возврата брошенной корзины)

**Цель:** Вернуть пользователя, не завершившего покупку, увеличить конверсию.

| Этап | Триггер                       | Действие                          | Система   | Задержка | Условие ветвления            | Цель            | Источник данных                                       |
| ---- | ----------------------------- | --------------------------------- | --------- | -------- | ---------------------------- | --------------- | ----------------------------------------------------- |
| 1    | add_to_cart → no purchase 24h | Email с напоминанием о корзине    | ESP       | 24 часа  | -                            | Возврат клиента | Email_Tech_Setup.md → Раздел 3 → Follow-up            |
| 2    | email_not_opened              | Push / SMS напоминание            | CRM / ESP | 2 дня    | Если не открыл → второй шанс | Второй шанс     | CRM_Integration_Map.md → Раздел 3                     |
| 3    | email_clicked                 | Персонализированное предложение   | ESP       | 0 минут  | Если кликнул → горячий лид   | Конверсия       | Email_Tech_Setup.md → Раздел 3 → Newsletter           |
| 4    | purchase                      | Завершение цепочки, благодарность | ESP       | 0 минут  | Если купил → завершение      | Конверсия       | Email_Tech_Setup.md → Раздел 3 → System/Transactional |

**Требуется ручная работа:** Да (человек должен создать воронку Abandoned Cart Flow в ESP, настроить триггеры, условия ветвления, связать с аналитикой, протестировать).

**Последовательность действий для человека:**

1. Создать воронку Abandoned Cart Flow в ESP.
2. Настроить триггер "add_to_cart" из Analytics_Tracking_Spec.md → Раздел 2 (через webhook или API).
3. Настроить условие "no purchase 24h" (отслеживание через Analytics_Tracking_Spec.md → Раздел 2).
4. Добавить шаблоны писем из Email_Tech_Setup.md → Раздел 3 (Follow-up, Newsletter, System/Transactional).
5. Настроить условия ветвления (открыл/не открыл, кликнул/не кликнул, купил/не купил).
6. Настроить задержки (24 часа для первого напоминания, 2 дня для второго).
7. Настроить интеграцию с аналитикой для отслеживания события "purchase" (согласно Analytics_Tracking_Spec.md → Раздел 2).
8. Протестировать воронку: добавить товар в корзину, подождать 24 часа, проверить отправку письма, проверить завершение цепочки при покупке.
9. Предоставить результаты тестирования ИИ-агенту для фиксации в разделе 7.

#### 4.4. Post-Purchase Flow (Воронка после покупки)

**Цель:** Повысить лояльность, получить отзывы, стимулировать повторные покупки.

| Этап | Триггер       | Действие                              | Система   | Задержка | Условие ветвления             | Цель              | Источник данных                                                   |
| ---- | ------------- | ------------------------------------- | --------- | -------- | ----------------------------- | ----------------- | ----------------------------------------------------------------- |
| 1    | purchase      | Email благодарности                   | ESP       | 0 минут  | -                             | Лояльность        | Email_Tech_Setup.md → Раздел 3 → System/Transactional             |
| 2    | email_opened  | Добавление в сегмент "Active Clients" | ESP + CRM | 0 минут  | Если открыл → активный клиент | Сегментация       | Email_Tech_Setup.md → Раздел 4, CRM_Integration_Map.md → Раздел 3 |
| 3    | через 14 дней | Follow-up с опросом/отзывом           | ESP       | 14 дней  | -                             | Отзывы            | Email_Tech_Setup.md → Раздел 3 → Follow-up                        |
| 4    | после 30 дней | Email с апселлом/кросс-селлом         | ESP       | 30 дней  | -                             | Повторная продажа | Email_Tech_Setup.md → Раздел 3 → Newsletter                       |

**Требуется ручная работа:** Да (человек должен создать воронку Post-Purchase Flow в ESP, настроить триггеры, задержки, связать с CRM, протестировать).

**Последовательность действий для человека:**

1. Создать воронку Post-Purchase Flow в ESP.
2. Настроить триггер "purchase" из Analytics_Tracking_Spec.md → Раздел 2 (через webhook или API).
3. Добавить шаблоны писем из Email_Tech_Setup.md → Раздел 3 (System/Transactional, Follow-up, Newsletter).
4. Настроить задержки (0 минут для благодарности, 14 дней для опроса, 30 дней для апселла).
5. Настроить интеграцию с CRM для добавления в сегмент "Active Clients" (согласно CRM_Integration_Map.md → Раздел 3, Email_Tech_Setup.md → Раздел 4).
6. Протестировать воронку: симулировать покупку, проверить отправку благодарности, проверить добавление в сегмент, проверить отправку follow-up писем.
7. Предоставить результаты тестирования ИИ-агенту для фиксации в разделе 7.
