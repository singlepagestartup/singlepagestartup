##### 4.2. Process управления изменениями (Change Control Process)

**1. Категоризация изменений:**

- **Type 1 (Критические):** Влияние на ROMI > 10%, требует перезапуска спринта
- **Type 2 (Значительные):** Влияние на ROMI 5-10%, требует пересмотра текущего спринта
- **Type 3 (Минорные):** Влияние на ROMI < 5%, может быть внедрено в текущий спринт

**2. Change Request Flow:**

```
Запрос изменения → Оценка влияния на ROMI/сроки/бюджет →
Согласование с CMO/Project Manager →
Обновление Project Plan →
Коммуникация с командой →
Внедрение в backlog
```

**3. Критерии принятия изменений:**

- ROMI Impact Score ≤ 5 (для срочных изменений)
- Availability of buffer resources
- Client approval for budget/timeline impact
- No critical conflicts with current sprint goals
