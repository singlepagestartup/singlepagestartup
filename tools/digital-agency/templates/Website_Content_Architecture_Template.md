## SYSTEM FILE: Website_Content_Architecture_Template.md

### PURPOSE: Система правил для ИИ-агентов по созданию архитектуры контента сайта, которая обеспечивает достижение бизнес-целей через правильное расположение информационных блоков на страницах.

### OWNER: Content Architect

### STRUCTURE: Определяет структуру сайта, контентные блоки для каждой страницы, их связь с этапами воронки продаж и промпты для ИИ-генерации контента.

---

### INSTRUCTIONS FOR USE:

- Content Architect заполняет все разделы на основе анализа **Content_Framework.md**, **Marketing_Strategy.md**, **Brand_Book.md** и **Company_Presentation.md**
- Все поля, помеченные как **[ОБЯЗАТЕЛЬНО]**, должны быть заполнены
- Если данные отсутствуют или требуют уточнения, указывать "Требует уточнения" с указанием, что именно нужно уточнить
- Раздел 1 должен содержать извлечённые данные из маркетинговой стратегии
- Раздел 2 должен содержать детальную структуру сайта с контентными блоками
- Раздел 3 должен содержать связь блоков с этапами воронки продаж
- Раздел 4 должен содержать промпты для ИИ-генерации контента страниц

**[ОБЯЗАТЕЛЬНО]** Определить контекст сайта и его связь с бизнес-целями.

**Инструкция:** Все данные извлекаются из **Marketing_Strategy.md**, **Content_Framework.md** и **Company_Presentation.md**.

- **Бизнес-цели сайта:** [Из Marketing_Strategy.md. Например: "Генерация лидов, увеличение узнаваемости бренда, поддержка продаж"]

- **Целевая аудитория:** [Из Marketing_Strategy.md. Например: "Владельцы малого бизнеса (1-10 сотрудников), менеджеры по маркетингу"]

#### 3. Связь контентных блоков с этапами воронки

**[ОБЯЗАТЕЛЬНО]** Описание того, как каждый контентный блок работает на этапы воронки.

- **Awareness (Внимание):**

  - Hero-блок на главной: "Привлекает внимание через яркий заголовок с проблемой клиента"
  - Блок проблемы: "Вызывает узнавание и показывает понимание боли аудитории"

- **Interest (Интерес):**

  - Блок решения: "Создает интерес к продукту через демонстрацию ценности"
  - Страница "О компании": "Повышает доверие через прозрачность и экспертизу"

- **Desire (Желание):**

  - Преимущества: "Усиливает желание через конкретные выгоды и цифры"
  - Кейсы и отзывы: "Создает желание через социальное доказательство"

- **Action (Действие):**
  - CTA-кнопки: "Направляют к конкретному действию с ясным призывом"
  - Формы контактов: "Облегчают процесс связи с компанией"

#### 4. Промпты для ИИ-генерации контента

**[ОБЯЗАТЕЛЬНО]** Готовые промпты для ИИ-генерации контента страниц.

##### 4.1. Шаблон промпта для главной страницы

```
Создай контент для главной страницы сайта компании [название компании]:

КОНТЕКСТ:
- Целевая аудитория: [из раздела 1]
- Тон коммуникации: [из раздела 1]
- Бизнес-цели: [из раздела 1]

СТРУКТУРА (из раздела 2.1):
1. Hero-блок (заголовок, подзаголовок, проблема)
2. Блок решения (описание, преимущества)
3. Преимущества (3-5 пунктов)
4. Социальное доказательство (кейсы, отзывы)
5. CTA (призыв к действию)

ЭТАПЫ ВОРОНКИ:
- Awareness: hero-блок, проблема
- Interest: решение
- Desire: преимущества, социальное доказательство
- Action: CTA

Создай тексты для каждого блока, учитывая тон коммуникации и целевую аудиторию.
```

##### 4.2. Шаблон промпта для страницы продукта

```
Создай контент для страницы продукта [название продукта]:

КОНТЕКСТ:
- Продукт: [описание из раздела 2.3]
- Целевая аудитория: [из раздела 1]
- Ключевые преимущества: [из раздела 2.3]

СТРУКТУРА (из раздела 2.3):
1. Описание продукта
2. Преимущества (3-5 пунктов с цифрами)
3. Цены/тарифы
4. FAQ (5-7 вопросов)
5. CTA (пробный период, консультация)

ЭТАПЫ ВОРОНКИ:
- Interest: описание продукта
- Desire: преимущества, цены
- Action: CTA, FAQ

Создай убедительные тексты для каждого раздела.
```

##### 4.3. Шаблон промпта для страницы кейсов

```
Создай контент для страницы кейсов компании:

КОНТЕКСТ:
- Отрасль: [из Marketing_Strategy.md]
- Тип клиентов: [из раздела 1]
- Ключевые результаты: [из Company_Presentation.md]

СТРУКТУРА (из раздела 2.4):
1. Выборка кейсов (фильтры)
2. Детальный кейс (проблема → решение → результат)
3. Цифры и метрики успеха
4. Отзывы клиентов
5. CTA (получить консультацию)

ЭТАПЫ ВОРОНКИ:
- Interest: выборка кейсов
- Desire: детальный кейс, отзывы
- Action: CTA

Создай контент, который доказывает эффективность через реальные примеры.
```

**Важно для ИИ-генерации:**

- Все промпты должны быть готовы для прямого использования в ИИ-моделях
- Промпты должны содержать все необходимые параметры для генерации качественного контента
- Связь с Content_Framework.md должна быть указана для использования правильных текстов
