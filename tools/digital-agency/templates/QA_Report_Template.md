## SYSTEM FILE: QA_Report_Template.md

### PURPOSE: Шаблон отчета о комплексном тестировании всех технических систем проекта (сайта и мобильного приложения). Документ фиксирует результаты проверки производительности, функциональности, безопасности и доступности для обеспечения готовности к следующим этапам (аналитика и SEO).

### OWNER: Chief Technology Officer (CTO)

### STRUCTURE: Определяет результаты проверки производительности (Core Web Vitals, время ответа API, производительность приложения), функциональности (формы, интеграции, навигация), безопасности (SSL, API, права доступа) и доступности (WCAG) для сайта и мобильного приложения.

---

### INSTRUCTIONS FOR USE:

- CTO заполняет все разделы на основе выполненной проверки всех технических систем проекта (сайта из Website_Deployment_Report.md и мобильного приложения из App_Development_Report.md).
- **[КРИТИЧЕСКИ ВАЖНО]** Все данные должны отражать реально выполненную проверку. НЕ указывать непроверенные функции или системы.
- Все поля, помеченные как **[ОБЯЗАТЕЛЬНО]**, должны быть заполнены
- Если обнаружены проблемы, указывать их конкретно с описанием, уровнем критичности и рекомендациями по исправлению
- Все технические параметры должны быть измеримыми и проверяемыми
- Ссылки на исходные документы (Website_Deployment_Report.md, App_Development_Report.md, Tech_Setup_Plan.md, Integration_Map.md) должны быть указаны для каждой проверки
