# Архитектура контента

## 1. Принципы создания текстов

**Подход на основе данных:**

- Каждый текст работает на конкретный показатель: увеличение возврат инвестиций, снижение стоимости привлечения, увеличение ценности, увеличение конверсии, увеличение удержания
- Конкретные цифры вместо общих фраз ("экономия 30%" вместо "значительная")

**Психологические принципы:**

- Социальное доказательство (кейсы, отзывы, статистика)
- Дефицит (ограничения по времени/количеству)
- Авторитет (экспертность, партнёрства)
- Избегание потерь (акцент на потерях от бездействия)
- Выгода против функции (фокус на выгодах)
- Якорение (конкретные цифры как якоря)

**Структура связки (контентная роль каждого шага):**

- **Охват:** Привлечь внимание → показать экспертность → дать причину подписаться
- **Слежение:** Дать ценность → построить доверие → подвести к офферу
- **Продажа:** Снять возражения → показать результат → призвать к действию

**Воронка внутри каждого шага:**

- **Проблема + любопытство** → **Решение + доверие** → **Выгоды + призыв к действию**

## 2. Правила для форматов

> Каждый формат обслуживает конкретный шаг связки:
>
> - **Охват:** VK Клипы, Яндекс Дзен, VC.ru, YouTube Shorts, SEO-статьи (привлечь + дать причину подписаться)
> - **Слежение:** Telegram-канал, VK-сообщество, Email-рассылка (прогреть + подвести к офферу)
> - **Продажа:** Посадочные страницы, формы, консультации (конвертировать)

### Сайт / Посадочные страницы (шаг: Продажа)

- Главный экран: Проблема + решение + призыв к действию (макс 3 сек)
- Структура PAS (Проблема-Усиление-Решение)
- Заголовки: выгода или цифра
- Обязательно: социальное доказательство под каждым предложением

### VK / YouTube / Дзен / Видео (шаг: Охват)

- **Структура:** Крючок → Проблема → Решение → Призыв подписаться
- **VK Клипы / YouTube Shorts:** Крючок (3 сек) → Проблема (5 сек) → Решение (7 сек) → Призыв подписаться (3 сек)
- **VK Сторис:** 3 слайда (Вопрос → Факт → «Подробнее в канале»)
- **Яндекс Дзен видео:** Крючок (5 сек) → Разбор проблемы → Решение → «Подписывайтесь» (до 3 мин)
- **Обязательно:** В каждом контенте — причина перейти в канал слежения (лидмагнит, продолжение, бонус)
- Хештеги для охвата алгоритмами

### VC.ru / Хабр / TenChat (шаг: Охват)

- **Структура:** Заголовок с результатом → Проблема отрасли → Решение + кейс → Призыв подписаться
- **Длина:** 1500-4000 слов, подзаголовки каждые 300-500 слов
- **Обязательно:** Ссылка на Telegram-канал / лидмагнит в конце статьи
- **Особенности VC.ru:** Экспертные кейсы, разборы, аналитика рынка
- **Особенности Хабр:** Технические статьи, туториалы, обзоры инструментов
- **Особенности TenChat:** Деловой контент, нетворкинг, короткие экспертные посты

### Telegram-канал / VK-сообщество / Email (шаг: Слежение)

- **Приветствие:** Выдать лидмагнит + показать ценность канала
- **Прогрев:** Полезный контент + кейсы + экспертность
- **Оффер:** Ограничение + выгода + призыв записаться/купить
- **Интервал:** 1-3 дня между постами/письмами
- **Обязательно:** Регулярные посты с оффером (каждый 5-7 пост — продающий)
- **Telegram:** Посты с форматированием, опросы, голосовые, кружочки
- **VK-сообщество:** Посты, статьи, обсуждения, рассылки через Senler/VK Рассылки
- **Email:** Тема (проблема или выгода) + персонализация + польза + призыв к действию

### SEO-статьи / Яндекс Дзен статьи (шаг: Охват)

- **Введение:** Показатели + обещание конкретики
- **Основное:** Проблемы → Решения с возвратом инвестиций → Демонстрация
- **Длина:** 1500-3000 слов с подзаголовками каждые 300 слов
- **SEO (Яндекс/Google):** Ключевое слово в заголовке + H1 + мета
- **Яндекс Дзен:** Привлекательный заголовок, обложка, дочитываемость > 40%
- **Обязательно:** Форма подписки / лидмагнит в статье (переход к шагу Слежение)

## 3. Стиль коммуникации

| Параметр    | Требование                 | Пример                                    |
| ----------- | -------------------------- | ----------------------------------------- |
| **Стиль**   | Экспертный, но дружелюбный | "Автоматизация снижает затраты на 30%"    |
| **Эмоции**  | Уверенность, спокойствие   | "Освободи время для стратегии"            |
| **Лексика** | Профессиональная, понятная | "Платформа автоматизирует"                |
| **Цифры**   | Конкретные, проверяемые    | "Экономия 2.5 млн ₽/год"                  |
| **Длина**   | Короткие абзацы            | Заголовок 5-8 слов, абзац 2-3 предложения |

**Запрещено:** Сленг без пояснения, пассивный залог, общие фразы, превосходные степени без доказательств, технические термины без выгоды.

## 4. Формулы генерации

### Универсальная формула заголовка

```
[Проблема клиента] + [Конкретная цифра] + [Вопрос/Решение] = Заголовок
```

### Формула поста для соцсетей

```
[Крючок с показателем] + [Проблема] + [Решение с выгодой] + [Призыв к действию] + [Хештеги] = Пост
```

### Формула статьи

```
[Заголовок с результатом] + [Введение с цифрами] + [Проблема с данными] + [Решение] + [Кейс с показателями] + [Вывод с призывом к действию] = Статья
```

### Формула email

```
[Тема: проблема/выгода] + [Персонализация] + [Заголовок] + [Польза] + [Призыв к действию] + [Подпись] = Письмо
```

## 5. Инструкция для ИИ

**Входные данные:**

- `project/content_framework.md` (правила)
- `project/marketing_strategy.md` (показатели, стиль, позиционирование бренда)
- `project/client_brief.md` (данные проекта)

**Алгоритм:**

1. Определи, для какой связки и какого шага создаётся контент (Охват / Слежение / Продажа)
2. Проанализируй входные документы (извлеки показатели, стиль коммуникации, данные)
3. Примени правила из `content_framework.md` (формат, структуру, формулу)
4. Убедись, что контент содержит переход к следующему шагу связки

**Формат ответа:**

- **Связка и шаг** (Связка #1, шаг: Охват)
- **Цель текста** (какой показатель увеличивает)
- **Переход к следующему шагу** (какой CTA ведёт дальше по связке)
- **Использованные принципы** (Социальное доказательство, Дефицит)
- **Ожидаемый результат** (прогноз конверсии на следующий шаг)
