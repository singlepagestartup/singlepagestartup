# Архитектура контента

## 1. Принципы создания текстов

**Подход на основе данных:**

- Каждый текст работает на конкретный показатель в порядке приоритета: **сначала масштаб** (охваты, просмотры, вовлечённость, подписки, клики), **затем конверсия при достаточном объеме данных** (увеличение возврата инвестиций, снижение стоимости привлечения, увеличение ценности, увеличение удержания)
- Конкретные цифры вместо общих фраз ("экономия 30%" вместо "значительная")

**Психологические принципы:**

- Социальное доказательство (кейсы, отзывы, статистика)
- Дефицит (ограничения по времени/количеству)
- Авторитет (экспертность, партнёрства)
- Избегание потерь (акцент на потерях от бездействия)
- Выгода против функции (фокус на выгодах)
- Якорение (конкретные цифры как якоря)

**Структура связки (контентная роль каждого шага):**

- **Охват:** Привлечь внимание → показать экспертность → дать причину подписаться
- **Слежение:** Дать ценность → построить доверие → подвести к офферу
- **Продажа:** Снять возражения → показать результат → призвать к действию

**Воронка внутри каждого шага:**

- **Проблема + любопытство** → **Решение + доверие** → **Выгоды + призыв к действию**

## 6. Формулы генерации

### Универсальная формула заголовка

```
[Проблема клиента] + [Конкретная цифра] + [Вопрос/Решение] = Заголовок
```

Примеры:

- "Как увеличить продажи на 40% за 30 дней"
- "5 ошибок, которые убивают вашу конверсию"
- "Почему 90% стартапов провалились с рекламой"

### Формула поста для соцсетей

```
[Крючок] + [Проблема] + [Решение] + [CTA] = Пост
```

### Формула статьи

```
[Заголовок с результатом] + [Введение с обещанием] + [Проблема] + [Решение пошагово] + [Кейс] + [CTA] = Статья
```

### Формула email

```
[Тема с интригой] + [Персонализация] + [Польза] + [CTA] + [P.S.] = Письмо
```

### Формула видео

```
[Крючок 0-3 сек] + [Проблема 3-8 сек] + [Решение 8-40 сек] + [CTA 40-60 сек] = Видео
```

### Формула продающей статьи/видео (VSL — Video Sales Letter)

> **Назначение:** Для создания длинных продающих материалов (статьи 1000-1500 слов, VSL 15-25 минут / 2000-3000 слов), которые прогревают холодную аудиторию и приводят к первой покупке.

**Структура из 10 обязательных элементов:**

#### 1. Хук (Захват внимания)

**Цель:** Остановить скролл, зацепить внимание за 3 секунды.

**Инструменты:**

- Боль: «Почему большинство [ЦА] работают в 3 раза больше, а зарабатывают в 2 раза меньше?»
- Любопытство: «3 ошибки, которые убивают 90% ваших результатов в [ниша]»
- Несправедливость: «Вот почему [обычная стратегия] больше не работает»

**Правила:**

- Конкретные цифры вместо общих фраз
- Вопрос или провокационное утверждение
- Прямое попадание в боль ЦА

#### 2. Диагноз проблемы

**Цель:** Показать, что вы понимаете их ситуацию лучше, чем они сами (чтобы кивали головой).

**Как:**

- Опиши боль, с которой они сталкиваются каждый день
- Покажи глубину проблемы (последствия, упущенные возможности)
- Усиль: «и это не ваша вина, система устроена так, что…»

**Правила:**

- Конкретные детали (не «трудно», а «тратите 5 часов в день на...»)
- Эмоциональный резонанс (фрустрация, усталость, разочарование)
- Снятие вины с клиента

#### 3. Переворот (Новый взгляд)

**Цель:** Дать инсайт, который меняет понимание проблемы. Люди должны почувствовать: «Ага, вот почему у меня не получалось».

**Инструменты:**

- Новый механизм: «Дело не в [то, что они думали], а в [истинная причина]»
- Главная ошибка: «Большинство делают X, но это устаревший подход»
- Скрытая причина: «Реальная проблема не на поверхности, она в...»

**Пример:**
«Дело не в том, что вы мало работаете. Дело в том, что вы используете старый инструмент — а он больше не приносит результата».

**Правила:**

- Должен быть неожиданным, но логичным
- Подводит к вашему решению как единственному правильному
- Создаёт момент «ага-эффекта»

#### 4. Решение (Уникальный механизм)

**Цель:** Представить продукт не как «курс/товар/услугу», а как механизм, который устраняет корневую проблему.

**Как:**

- Дай название механизму (например: «Контент-Завод», «Система 3-х воронок»)
- Покажи, КАК это работает (не что это, а как устраняет проблему)
- Свяжи с переворотом: «Поэтому мы создали [механизм], который...»

**Пример:**
«Мы создали систему, которая автоматизирует 90% процессов и экономит вам 20-30 часов в неделю. Мы называем её "Контент-Завод"».

**Правила:**

- Фокус на механизме, а не на продукте
- Конкретные цифры результата
- Связь с уникальным инсайтом из шага 3

#### 5. Доказательства

**Цель:** Показать, что это реально работает (социальное доказательство).

**Формат:**

- Кейсы клиентов (с цифрами: «За 2 недели увеличил конверсию с 2% до 7%»)
- Личная история (как вы сами нашли это решение)
- Скриншоты, графики, отзывы
- Даже если кейсов мало — используй личный опыт/историю

**Правила:**

- Конкретные цифры и сроки
- Релевантность к ЦА (похожая ситуация)
- Минимум 1-3 примера

#### 6. Оффер (Что именно получат)

**Цель:** Дать конкретику, что клиент получит и какого результата достигнет.

**Формула:** «Вы получаете → делаете → достигаете»

**Структура:**

- Перечисли компоненты продукта (модули, инструменты, доступы)
- Фокус на результате, а не на объёме («не 10 часов видео», а «система, которая принесёт первых клиентов за 14 дней»)
- Пропиши конкретику: «Вы получите X, Y, Z»

**Правила:**

- Результат > процесс
- Конкретные выгоды по пунктам
- Связь с их болью из шага 2

#### 7. Цена + Сравнение

**Цель:** Показать контраст и создать ощущение выгоды.

**Как:**

- Якорение: «Обычно это стоит X» или «Альтернативы заберут у вас месяцы и тысячи долларов»
- Разбивка: «Это всего 30₽ в день» (если применимо)
- Контраст: «Сейчас вы можете получить всё за [цена]»
- Сравнение: «Один проект окупит вложения в 10 раз»

**Правила:**

- Сначала контекст (дорогая альтернатива), потом цена
- Показать ценность, а не дешевизну
- Якорь должен быть реалистичным

#### 8. Снятие риска (Гарантия)

**Цель:** Минимизировать страх покупки, убрать последний барьер.

**Формат:**

- Гарантия возврата: «Если за 3 дня поймёте, что это не ваше — мы вернём деньги»
- Гарантия результата: «Если не получите X, мы работаем бесплатно до результата»
- Обратная гарантия: «Вы ничего не теряете — риск на нас»

**Правила:**

- Чёткие условия (срок, критерии)
- Простой процесс возврата
- Уверенный тон (мы верим в продукт)

#### 9. Призыв к действию (CTA)

**Цель:** Дать чёткую, понятную инструкцию, что делать прямо сейчас.

**Формат:**

- Одно действие, без распыления
- Глаголы действия: «Нажмите», «Получите», «Присоединяйтесь», «Запишитесь»
- Повторить несколько раз в тексте/видео (в начале, середине, конце)

**Правила:**

- Конкретный шаг (не «свяжитесь», а «нажмите кнопку ниже и заполните форму за 30 секунд»)
- Минимум трения (чем проще, тем лучше)
- Напоминание о выгоде: «Нажмите, чтобы получить [результат]»

#### 10. Усиление финала

**Цель:** Создать срочность и закрыть эмоционально, чтобы человек действовал сейчас.

**Инструменты:**

- Срочность: «Доступ открыт только до [дата]»
- Ограничение: «Осталось 5 мест» / «Цена вырастет через 3 дня»
- Эмоциональный выбор: «Вы можете продолжать тратить время, или уже сегодня запустить новую систему и освободить 20 часов в неделю»

**Правила:**

- Честная срочность (не фейковые таймеры)
- Эмоциональный контраст: «путь боли vs путь результата»
- Последнее напоминание о главной выгоде

---

**Итоговая формула VSL:**

```
[1. Хук] → [2. Диагноз проблемы] → [3. Переворот] → [4. Решение] → [5. Доказательства] → [6. Оффер] → [7. Цена] → [8. Гарантия] → [9. CTA] → [10. Усиление] = Продающая статья/VSL
```

**Стиль написания VSL:**

- Живой, разговорный язык (как будто объясняешь другу)
- Короткие фразы и абзацы (для динамики)
- Эмоциональные крючки, вопросы, метафоры
- Избегай канцелярита и клише
- Для VSL-видео — текст в стиле устной речи

**Объём:**

- Статья: 1000-1500 слов
- VSL-видео: 15-25 минут (2000-3000 слов)

## 10. Инструкция для ИИ-генератора контента

> **Важно:** Этот файл содержит УНИВЕРСАЛЬНЫЕ правила коммуникации бренда. Конкретные задачи по связкам (что создать, для какой платформы, какие темы) находятся в `project/project_plan.md` (раздел 2: Задачи для AI-агентов).

### Алгоритм работы AI-генератора:

**Шаг 1. Получи конкретную задачу**

- Прочитай `project/project_plan.md` → раздел 2 (Задачи для AI-агентов)
- Найди свою задачу (например: "AI-001: Сценарии для коротких видео")
- Извлеки: связку, платформу, канал, темы, формат вывода

**Шаг 2. Изучи универсальные правила бренда (этот файл)**

- **Раздел 1:** Принципы создания текстов (психология, воронка)
- **Раздел 3:** Голос и тон бренда (как звучит бренд)
- **Раздел 4:** Визуальная идентичность (как выглядит контент)
- **Раздел 6:** Формулы генерации (структуры для разных форматов)
- **Разделы 7-9:** Лидмагниты, офферы, скрипты (при необходимости)

**Шаг 3. Прочитай технические требования платформы**

- `platforms/[платформа].md` — форматы, разрешения, длительность, алгоритмические особенности

**Шаг 4. Изучи контекст проекта**

- `project/client_brief.md` — ЦА, продукт, боли, конкуренты
- `project/marketing_strategy.md` — позиционирование, офферы, лидмагниты для связки

**Шаг 5. Создай контент по типу**

**Для обычного контента (посты, статьи, видео):**

- Примени формулу из раздела 6 (для соответствующего формата)
- Используй голос бренда из раздела 3
- Соблюди технические требования платформы
- Включи переход на следующий шаг связки (CTA)

**Для продающих материалов (VSL, лендинги, вебинары):**

- Используй структуру VSL (10 шагов) из раздела 6
- Перед созданием задай уточняющие вопросы (если данных недостаточно):
  - Какие конкретные боли ЦА? (для диагноза)
  - Какой главный инсайт/переворот? (что они понимают неправильно)
  - Есть ли кейсы/доказательства с цифрами?
  - Какая гарантия предусмотрена?
  - Какое ограничение по времени/количеству?
- Работай итеративно: после первой версии предложи 2-3 варианта улучшений

**Шаг 6. Применяй продвинутые приёмы**

- **Few-shot prompting:** Создай 2-3 варианта хука/инсайта/CTA для выбора лучшего
- **Chain-of-thought:** Перед финальным текстом объясни, какой угол подачи выбрал и почему
- **Self-check:** После создания проверь по чек-листу (см. Шаг 7)
- **Iterative prompting:** Предложи 2 варианта финала (с акцентом на срочность и на эмоциональный выбор)

**Шаг 7. Проверь качество**

**Базовые критерии (для всех форматов):**

- ✓ Есть крючок (привлечение внимания)
- ✓ Есть проблема (резонанс с болью ЦА)
- ✓ Есть решение (конкретное предложение)
- ✓ Есть CTA (чёткий следующий шаг)
- ✓ Конкретные цифры вместо общих фраз
- ✓ Социальное доказательство (где применимо)
- ✓ Соответствие голосу бренда (раздел 3)
- ✓ Соответствие визуальной концепции (раздел 4)

**Дополнительные критерии для VSL:**

- ✓ Хук цепляет за 3 секунды (боль/любопытство/несправедливость)
- ✓ Есть инсайт/переворот (момент "ага")
- ✓ Решение представлено как уникальный механизм (не просто "курс")
- ✓ Есть конкретные доказательства с цифрами
- ✓ Оффер понятный (что получают → делают → достигают)
- ✓ Есть снятие риска (гарантия)
- ✓ Есть срочность/ограничение в финале
- ✓ CTA повторяется минимум 2-3 раза
- ✓ Эмоциональное закрытие (выбор: путь боли vs путь результата)

Следуй формату из промпта задачи в `project/project_plan.md`.
