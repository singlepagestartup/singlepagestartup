## SYSTEM FILE: Company_Presentation_Template.md

### PURPOSE: Создание текстового контента презентации компании, который следует логике управляемой воронки продаж, и описание графических элементов для слайдов, соответствующих Brand_Book.md.

### OWNER: Content Architect

### STRUCTURE: Определяет логику повествования презентации как путь слушателя по воронке AIDAAR, создает текстовый контент для каждого слайда и описывает графические элементы для ИИ-генерации.

---

### INSTRUCTIONS FOR USE:

- Content Architect заполняет все разделы на основе анализа **Content_Framework.md** и **Brand_Book.md**
- Все поля, помеченные как **[ОБЯЗАТЕЛЬНО]**, должны быть заполнены
- Если данные отсутствуют или требуют уточнения, указывать "Требует уточнения" с указанием, что именно нужно уточнить
- Раздел 1 должен содержать контекст и цели презентации на основе Content_Framework.md
- Раздел 2 должен содержать текстовый контент для каждого слайда, следующий логике воронки
- Раздел 3 должен содержать описание графических элементов для ИИ-генерации, соответствующих Brand_Book.md
- Раздел 4 должен содержать технические требования и рекомендации
- **КРИТИЧЕСКИ ВАЖНО:** Логика презентации должна четко прослеживать движение слушателя по воронке AIDAAR

#### 4. Технические требования

**[ОБЯЗАТЕЛЬНО]** Детальные технические требования для презентаций.

##### 4.1. Размеры и форматы слайдов

| Формат                       | Размер      | Соотношение сторон    | Формат файла       | Применение                                     |
| ---------------------------- | ----------- | --------------------- | ------------------ | ---------------------------------------------- |
| **Стандартная презентация**  | 1920×1080px | 16:9 (горизонтальный) | PPTX, PDF, Keynote | [Основной формат для презентаций]              |
| **Альтернативный формат**    | 1280×720px  | 16:9 (горизонтальный) | PPTX, PDF          | [Для быстрой загрузки или мобильных устройств] |
| **Вертикальная презентация** | 1080×1920px | 9:16 (вертикальный)   | PPTX, PDF          | [Для мобильных устройств или социальных сетей] |

##### 4.2. Технические характеристики

- **Разрешение:** [Например: "Минимум 72 dpi для экрана, 300 dpi для печати (если применимо)"]
- **Формат файлов:** [Например: "PPTX (PowerPoint) для редактирования, PDF для распространения"]
- **Вес файла:** [Например: "Не более 50MB для презентации с изображениями"]
- **Оптимизация изображений:** [Например: "Изображения должны быть оптимизированы (JPG 80% качество или PNG с прозрачностью), не более 500KB на изображение"]

##### 4.3. Требования к тексту

- **Читаемость:** [Например: "Минимальный размер текста: 14px для читаемости на проекторе"]
- **Контрастность:** [Например: "Минимальный контраст 4.5:1 для текста на фоне (WCAG)"]
- **Длина текста:** [Например: "Не более 7 строк текста на слайд для лёгкости восприятия"]

##### 4.4. Требования к анимациям и переходам

- **Анимации:** [Например: "Минимальные анимации, только для привлечения внимания к ключевым элементам (например, появление CTA-кнопки)"]
- **Переходы:** [Например: "Плавные переходы между слайдами (Fade или Slide), не более 0.5 секунды"]
- **Интерактивность:** [Например: "Кнопки должны быть кликабельными (если презентация в цифровом формате)"]

**Важно для дизайнеров:**

- Все технические требования должны быть указаны для правильной подготовки презентации
- Форматы должны соответствовать требованиям платформы (PowerPoint, Google Slides, Keynote)
