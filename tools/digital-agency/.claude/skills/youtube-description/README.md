# Skill: YouTube Description Generator

Автоматическое создание профессиональных описаний для длинных YouTube видео на основе транскрипции.

## Назначение

Создавать качественные описания для YouTube видео с:

- Структурированным текстом
- Таймкодами (15-20 меток)
- 5 ключевыми словами
- Соблюдением правил проекта и платформы

## Использование

### Базовая команда

```bash
/youtube-description path/to/transcription.txt
```

### Алиасы

```bash
/yt-desc path/to/transcription.txt
/youtube-desc path/to/transcription.txt
```

### Пример

```bash
/youtube-description project/content/video/podcasts/podcast_s_evgeniem_lvovym/content.txt
```

Результат будет сохранен в:

```
project/content/video/podcasts/podcast_s_evgeniem_lvovym/youtube_description.md
```

## Что делает skill

1. **Читает контекст:**

   - `platforms/youtube.md` — правила платформы
   - `project/content_framework.md` — стиль коммуникации
   - `project/README.md` — позиционирование проекта
   - Транскрипцию видео (указанный файл)

2. **Анализирует:**

   - Основную тему видео
   - Спикеров и их экспертизу
   - Ключевые темы и подтемы
   - Таймкоды важных моментов

3. **Создает описание:**

   - Краткое и развернутое описание
   - 15-20 таймкодов с описаниями
   - Список основных тем
   - Информация о спикерах
   - Объяснение ценности контента
   - CTA и ссылки
   - 5 ключевых слов в формате хештегов (без пробелов: #словоБезПробелов)
   - Финальная цитата

4. **Соблюдает правила:**
   - БЕЗ эмодзи (критично!)
   - БЕЗ markdown форматирования
   - Хештеги БЕЗ пробелов внутри (например: #доходнаяНедвижимость)
   - Стиль: экспертный + честный
   - Избегает запрещенных слов ("успешный успех", "гарантия" и т.д.)
   - Использует реальные цифры

## Структура результата

```
[Краткое описание]

0:00:19 - Приветствие

[CTA и ссылки]

#хештег1 #хештег2 #хештег3 #хештег4 #хештег5

[Финальная цитата]
```

## Требования к транскрипции

Skill работает с любыми текстовыми файлами, но оптимально:

- Формат: `.txt` или `.md`
- Таймкоды в формате: `XX:XX:XX.XXX` или `XX:XX:XX`
- Указаны имена спикеров
- Размер: любой (skill умеет работать с большими файлами)

## Автоматический контекст

Skill автоматически подгружает:

1. **platforms/youtube.md** — технические требования YouTube, лучшие практики
2. **project/content_framework.md** — стиль коммуникации, запрещенные слова
3. **project/README.md** — позиционирование, ЦА, УТП проекта

Тебе НЕ нужно вручную передавать эти файлы — skill сделает это сам.

## Обработка больших файлов

Если транскрипция превышает 25000 токенов, skill автоматически:

- Читает файл по частям (`Read` с `offset`/`limit`)
- Или использует `Grep` для извлечения ключевой информации

## Проверка результата

Перед публикацией убедись:

- ✅ Нет эмодзи
- ✅ Нет markdown (жирный, курсив, заголовки)
- ✅ Есть 15-20 таймкодов
- ✅ Ровно 5 ключевых слов в формате хештегов
- ✅ Хештеги БЕЗ пробелов (#доходнаяНедвижимость, а не "доходная недвижимость")
- ✅ ОСНОВНЫЕ ТЕМЫ структурированы (заголовок + краткое описание)
- ✅ Стиль соответствует проекту
- ✅ Файл сохранен в правильном месте

## Примечания

- Skill НЕ публикует видео — только создает описание
- Результат сохраняется локально в `youtube_description.md`
- Таймкоды автоматически преобразуются в YouTube формат
- Ключевые слова подбираются на основе контента и контекста проекта
