# Процесс работы агентства

## 01. Сбор информации о клиенте

**РОЛЬ:** Аккаунт-менеджер

**ЦЕЛЬ:** Сбор бизнес-информации о клиенте, формирование креативной спецификации и определение текущих активов по шаблону `client_brief_template.md`.

**ВХОДНЫЕ ДАННЫЕ:**

- Краткое описание клиента (чем занимается, продукт, цель обращения)

**РЕЗУЛЬТАТ:** Бриф клиента

**ШАБЛОН:** `client_brief_template.md`

---

### Алгоритм действий

#### Этап 1. Инициализация

1. При получении команды "Запусти этап 01" агент:
   - Проверяет, что шаблон `client_brief_template.md` доступен
   - Сообщает пользователю, что начнёт задавать вопросы для заполнения анкеты
   - Объясняет, что результат станет источником данных для проекта

#### Этап 2. Сбор информации

Задать вопросы по секциям 1-5 и 9:

1. Общая информация о клиенте
2. Бизнес-модель
3. Аудитория и проблемы
4. Цели проекта, показатели и бюджет
5. Конкуренты и преимущества
6. Текущие активы (сайт, CRM, соцсети, email-база, рекламные аккаунты)

На основе собранных данных заполнить секции 6-8 и 10: 6. Креативная спецификация (сообщение, эмоция, стиль коммуникации, воронка продаж, ключевые посылы) 7. Форматы и материалы (каналы, технические требования, приоритет запуска) 8. Критерии успеха (целевые метрики по каналам) 10. Аналитическая сводка (ключевые выводы и рекомендации)

#### Этап 3. Формирование результата

1. Собрать все ответы в структурированный текст, соответствующий шаблону
2. Сохранить результат в `project/client_brief.md`
3. Сгенерировать `project/README.md` по шаблону `templates/readme_template.md` — заполнить секции аккаунт-менеджера

## 02. Стратегия маркетинга

**РОЛЬ:** Маркетинговый директор

**ЦЕЛЬ:** Разработка стратегии маркетинга с гипотезами связок «Охват → Слежение → Продажа».

**ВХОДНЫЕ ДАННЫЕ:**

- бриф клиента (`project/client_brief.md`)

**РЕЗУЛЬТАТ:** Стратегия маркетинга

**ШАБЛОН:** `marketing_strategy_template.md`

---

### Алгоритм действий

#### Этап 1. Инициализация

1. Проверить наличие брифа клиента
2. Сообщить о начале разработки стратегии

#### Этап 2. Контекст и позиционирование бренда

1. Извлечь из брифа контекст и цели, зафиксировать в разделе 1 шаблона:
   - Миссия бренда, бизнес-цели, целевые показатели
2. Определить позиционирование бренда из брифа:
   - Позиционирование — влияние на жизненную ценность и удержание
   - Миссия — влияние на доверие
   - Ценности — влияние на удержание
   - Ценностное предложение — влияние на конверсию
3. Установить стиль коммуникации и ключевые сообщения
4. Зафиксировать в разделе 2 шаблона

#### Этап 3. Цифровая стратегия

1. Спроектировать минимум 5 связок «Охват → Слежение → Продажа»:
   - Использовать каталог площадок из `platforms/README.md` (все категории: видео, статьи, соцсети, подкасты, нишевые, реклама)
   - Рассматривать ВСЕ подходящие площадки, включая нестандартные для отрасли (Pinterest, Reddit, подкасты, Discord и т.д.)
   - Для каждой связки определить:
     - Канал охвата (где алгоритмы/реклама/поиск покажут нас аудитории)
     - Канал слежения (куда подписывается аудитория)
     - Механизм продажи (как подписчик становится клиентом)
     - Лидмагнит (причина подписаться)
     - Оффер (причина купить)
   - Приоритезировать связки
   - Детализировать каждую связку (охват, слежение, продажа с показателями)
2. Зафиксировать в разделе 3 шаблона

#### Этап 4. Формирование стратегии

1. Проверить заполненность всех разделов шаблона (1-3)
2. Сохранить результат в `project/marketing_strategy.md`
3. Обновить `project/README.md` по шаблону `templates/readme_template.md`: секции «Позиционирование» и «Связки»

## 03. План тестирования связок

**РОЛЬ:** Менеджер проектов

**ЦЕЛЬ:** Превращение связок из стратегии в план тестирования: гипотезы, конкретные задачи, задание для архитектора контента и критерии проверки.

**ВХОДНЫЕ ДАННЫЕ:**

- бриф клиента (`project/client_brief.md`)
- стратегия маркетинга (`project/marketing_strategy.md`)

**РЕЗУЛЬТАТ:** План тестирования связок

**ШАБЛОН:** `project_plan_template.md`

---

### Алгоритм действий

#### Этап 1. Инициализация

1. Проверить наличие брифа и стратегии маркетинга
2. Извлечь все 5 связок из стратегии с их приоритетами, лидмагнитами и офферами
3. Зафиксировать исходные данные в разделе 1 шаблона

#### Этап 2. Генерация гипотез

1. Для каждой связки сформулировать 3+ гипотезы:
   - Базовая гипотеза (ожидаемый сценарий)
   - Альтернативная гипотеза (другой подход к тому же каналу)
   - Смелая/нестандартная гипотеза (неочевидный формат, необычный лидмагнит, провокационный оффер)
2. Для каждой гипотезы определить:
   - Что именно тестируем (одна переменная)
   - Ожидаемый результат (конкретная метрика)
   - Способ проверки
3. Зафиксировать в разделе 2 шаблона

#### Этап 3. Задачи по связкам

1. Для каждой связки расписать конкретные задачи по трём шагам:
   - **Охват:** что создать (канал, контент, механизм охвата, CTA на слежение)
   - **Слежение:** что настроить (канал, лидмагнит, приветствие, контент-план прогрева)
   - **Продажа:** что подготовить (оффер, механизм приёма заявок, материалы для закрытия)
2. Задачи должны быть конкретными и выполнимыми (чек-лист)
3. Зафиксировать в разделе 3 шаблона

#### Этап 4. Задание для архитектора контента

1. Определить, какие правила и форматы нужны от этапа 04:
   - Для каждой связки × каждого шага: формат и требования к контенту
   - Приоритеты (какие связки описать первыми)
   - Особые требования (лидмагниты, офферы, серии писем)
2. Сформулировать как конкретное задание (что архитектор контента должен создать)
3. Зафиксировать в разделе 4 шаблона

#### Этап 5. Критерии проверки

1. Установить общие правила тестирования:
   - Минимальный срок теста (обычно 1 неделя)
   - Правило одной переменной
   - Порог эффективности (недельные охваты в просмотрах/визитах)
2. Для каждой связки определить:
   - Целевые охваты за неделю (например: "100K просмотров за неделю")
   - Целевая конверсия Охват→Слежение (например: "1% подписок от просмотров")
   - Целевая конверсия Слежение→Продажа (например: "5% заявок от подписчиков")
   - Решение при успехе (масштабировать)
   - Решение при провале (усилить звено / стоп)
3. Описать диагностику слабого звена (симптом → проблема → действие)
4. Примечание: стоимость клиента (CAC) считается скопом по всем связкам раз в неделю/месяц (общий расход / количество клиентов), так как сквозная аналитика не всегда доступна
5. Зафиксировать в разделе 5 шаблона

#### Этап 6. Последовательность запуска

1. Определить порядок запуска связок (по приоритету из стратегии)
2. Для каждого периода указать:
   - Действие (подготовка / запуск / анализ)
   - Какие связки
   - Ожидаемый результат
   - Условие перехода к следующему шагу
3. Зафиксировать в разделе 6 шаблона

#### Этап 7. Формирование плана

1. Проверить заполненность всех разделов шаблона (1-6)
2. Сохранить результат в `project/project_plan.md`
3. Обновить `project/README.md` по шаблону `templates/readme_template.md`: секция «Тестирование связок»

## 04. Архитектура контента

**РОЛЬ:** Архитектор контента

**ЦЕЛЬ:** Разработка правил контента для каждого шага связки «Охват → Слежение → Продажа» на основе задания из плана тестирования.

**ВХОДНЫЕ ДАННЫЕ:**

- план тестирования связок (`project/project_plan.md`) — задание для архитектора (раздел 4)
- стратегия маркетинга (`project/marketing_strategy.md`)
- бриф клиента (`project/client_brief.md`)

**РЕЗУЛЬТАТ:** Правила контента

**ШАБЛОН:** `content_framework_template.md`

---

### Алгоритм действий

#### Этап 1. Инициализация

1. Проверить наличие плана тестирования, стратегии и брифа
2. Извлечь задание для архитектора из `project/project_plan.md` (раздел 4: форматы, приоритеты, особые требования)
3. Сообщить о начале разработки правил создания контента

#### Этап 2. Принципы создания текстов

1. Извлечь связки из стратегии маркетинга
2. Определить подход на основе данных:
   - Каждый контент обслуживает конкретный шаг связки
   - Каждый контент содержит переход к следующему шагу
   - Конкретные цифры вместо общих фраз
3. Установить психологические принципы:
   - Социальное доказательство, дефицит, авторитет
   - Избегание потерь, выгода против функции

#### Этап 3. Правила для форматов (по шагам связки)

Разработать правила для каждого формата с привязкой к шагу связки:

1. **Охват** (VK Клипы, Дзен, VC.ru, YouTube, SEO, Яндекс Директ): Крючок — Проблема — Решение — Призыв подписаться
2. **Слежение** (Telegram, VK-сообщество, Email): Лидмагнит — Прогрев — Оффер
3. **Продажа** (лендинг, форма): структура PAS, заголовки с выгодой, призыв к действию
4. **Обязательно:** каждый контент охвата содержит CTA на слежение, каждый контент слежения содержит CTA на продажу

#### Этап 4. Стиль коммуникации

1. Установить параметры:
   - Стиль: экспертный, но дружелюбный
   - Эмоции: уверенность, спокойствие
   - Лексика: профессиональная, понятная
   - Цифры: конкретные, проверяемые
2. Описать запрещённые приёмы

#### Этап 5. Формулы генерации

Создать формулы для генерации:

1. Заголовок: Проблема + Цифра + Решение
2. Пост для соцсетей: Крючок + Проблема + Решение + Призыв к действию
3. Статья: Заголовок с результатом + Введение + Проблема + Решение + Кейс
4. Email: Тема (проблема или выгода) + Персонализация + Польза + Призыв к действию

#### Этап 6. Инструкция для агентов

1. Описать входные данные:
   - Правила контента
   - Стратегия маркетинга (связки, показатели, стиль)
   - Бриф клиента (данные проекта)
2. Установить алгоритм: определить связку и шаг → применить правила → убедиться в наличии CTA на следующий шаг
3. Установить формат ответа (связка, шаг, цель, переход, принципы, результат)

#### Этап 7. Формирование правил контента

1. Заполнить шаблон всеми разделами
2. Убедиться, что все формулы и правила конкретны
3. Сохранить результат в `project/content_framework.md`
4. Обновить `project/README.md` по шаблону `templates/readme_template.md`: статус артефакта

## 05. Анализ эффективности

**РОЛЬ:** Аналитик

**ЦЕЛЬ:** Анализ эффективности связок «Охват → Слежение → Продажа» с выявлением слабых звеньев и гипотезами оптимизации.

**ВХОДНЫЕ ДАННЫЕ:**

- стратегия маркетинга (`project/marketing_strategy.md`)
- бриф клиента (`project/client_brief.md`)
- правила контента (`project/content_framework.md`)
- данные из рекламных кабинетов, CRM, аналитики (предоставляет пользователь)

**РЕЗУЛЬТАТ:** Отчёт об эффективности

**ШАБЛОН:** `performance_analysis_report_template.md`

---

### Алгоритм действий

#### Этап 1. Инициализация

1. Проверить наличие всех входных файлов (marketing_strategy.md, client_brief.md, content_framework.md)
2. Извлечь из стратегии список активных связок и каналов
3. Указать период отчёта (обычно последняя неделя или месяц)

#### Этап 2. Запрос данных у пользователя

Сформировать список необходимых данных с инструкциями по получению. Для каждого источника указать:

- Где получить данные (платформа → раздел → экспорт)
- Какие данные нужны (метрики, период)
- В каком формате сохранить (CSV, Excel, скриншот)
- Куда сохранить (директория `project/analytics/`)

**Инструкции по сбору данных:**

Для каждой платформы, используемой в связках проекта, используй детальные инструкции из соответствующего файла `platforms/`:

- **Яндекс** (Метрика, Дзен, Директ): `platforms/yandex.md` → раздел "Сбор данных для отчета"
- **VK** (Статистика, Реклама): `platforms/vk.md` → раздел "Сбор данных для отчета"
- **Telegram** (TGStat): `platforms/telegram.md` → раздел "Сбор данных для отчета"
- **Instagram** (Meta Business Suite): `platforms/instagram.md` → раздел "Сбор данных для отчета"
- **TikTok** (Analytics): `platforms/tiktok.md` → раздел "Сбор данных для отчета"
- **YouTube** (Analytics): `platforms/youtube.md` → раздел "Сбор данных для отчета"
- **Email** (сервисы рассылок): `platforms/email.md` → раздел "Сбор данных для отчета"

**Дополнительно (не привязано к платформе):**

```
### Данные из CRM (amoCRM/Bitrix24)
Инструкция: CRM → Сделки → Фильтр по периоду → Экспорт → Excel/CSV
Период: [начало] - [конец]
Метрики: лиды, сделки, источники, статусы
Сохранить в: project/analytics/crm_deals.csv
```

После формирования списка — **остановиться и дождаться, пока пользователь загрузит файлы в `project/analytics/`**.

#### Этап 3. Проверка наличия данных

1. Проверить директорию `project/analytics/` на наличие файлов
2. Если файлов нет — запросить их у пользователя повторно
3. Если файлы есть — перейти к анализу

#### Этап 4. Извлечение данных из файлов

1. Прочитать файлы из `project/analytics/`:
   - Открыть Excel/CSV файлы (pandas, csv)
   - Распознать данные из скриншотов (если нужно)
   - Извлечь метрики: охваты, клики, подписки, конверсии, расходы
2. Свести данные в общую таблицу по связкам
3. Зафиксировать проблемы (если данных нет или файлы повреждены)

#### Этап 5. Методология показателей

1. Описать формулы расчёта (используемые в отчете):
   - Охват за неделю = суммарные просмотры/визиты за период
   - ER (Engagement Rate) = (Лайки + Комментарии + Репосты) / Просмотры × 100%
   - Конверсия = Конверсии / Сессии × 100%
   - Кликабельность (CTR) = Клики / Показы × 100%
   - Возврат инвестиций (ROI) = (Доход − Расход) / Расход × 100% (скопом)
   - Стоимость привлечения (CAC) = Расход / Клиенты (скопом)

#### Этап 6. Показатели относительно целей

1. Сравнить фактические показатели с целями из стратегии:
   - Возврат инвестиций: цель — факт — разница
   - Стоимость привлечения: цель — факт — разница
   - Жизненная ценность: цель — факт — разница
   - Конверсия: цель — факт — разница
   - Удержание: цель — факт — разница

#### Этап 7. Эффективность по связкам

1. Для каждой связки из стратегии свести показатели из файлов:
   - Охват за неделю (просмотры/визиты) — основной критерий эффективности
   - Конверсия Охват→Слежение (% подписавшихся)
   - Конверсия Слежение→Продажа (% купивших)
   - Количество клиентов (без привязки к CAC конкретной связки)
2. Определить слабое звено каждой связки (где наибольшие потери)
3. Указать причину потерь на слабом звене
4. Примечание: CAC считается только скопом (суммарный расход недели / суммарные клиенты недели)

#### Этап 8. Воронка по шагам связки

1. Построить общую воронку из данных файлов:
   - Охват (суммарные просмотры/визиты)
   - Охват→Слежение (подписки/регистрации)
   - Слежение (активные подписчики)
   - Слежение→Продажа (заявки/консультации)
   - Продажа (закрытые сделки)
2. Указать главную точку потерь

#### Этап 9. Материалы и аудитории

1. Проанализировать контент охвата из данных (топ-10):
   - Просмотры, конверсия в подписку
   - Какие форматы/темы дают больше подписок
   - Соответствие правилам из content_framework.md (структура, CTA, переходы между шагами)
2. Проанализировать контент слежения из данных:
   - Какие посты/письма дают заявки
   - Соответствие формулам генерации из content_framework.md
3. Дать рекомендации по усилению слабого звена (включая обновление правил контента)

#### Этап 10. Выводы и гипотезы

1. Сформулировать выводы по связкам на основе данных:
   - Лучшая связка (масштабировать)
   - Слабые связки (остановить или усилить)
   - Новые связки для тестирования
2. Сформировать гипотезы оптимизации связок:
