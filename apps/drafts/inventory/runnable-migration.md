# Runnable Migration Matrix

Issue: `201`

This matrix is the Phase 1 audit trail for moving remaining `apps/drafts/runnable`
source material into the Storybook module catalog. The goal is not to preserve
runnable app structure. Each meaningful runnable component, page, layout,
context, editor, and admin support surface gets one planned outcome.

Outcome values:

- `migrated`: reusable presentation now has module-owned Storybook coverage or a host page recipe.
- `replaced`: an existing Storybook draft or static props pattern covers
  the reusable surface.
- `retained`: keep the runnable entry as a standalone prototype with a recorded
  reason.

Source aliases:

- `SP`: `apps/drafts/runnable/singlepage/admin-v2`
- `ST`: `apps/drafts/runnable/startup/singlepagestartup`
- `HTML_ADMIN`: `apps/drafts/runnable/singlepage/admin-panel-redesign-html`
- `HTML_BASIC`: `apps/drafts/runnable/singlepage/examples/basic-html`

Excluded from component migration rows: `node_modules`, lockfiles, `.DS_Store`,
package manager files, generated output, Figma Make attribution files, and
standalone build configuration. Runnable manifests are listed only to classify
each runnable entry.

## Ownership Rules

These destinations follow the module README boundaries:

| Area                                                                   | Owner                                                                                                                  |
| ---------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| Route/page recipes, layout composition, host widget slots              | `host.models.page`, `host.models.layout`, `host.models.widget`                                                         |
| Generic content sections, nav/footer, buttons, feature/content widgets | `website-builder.models.*` and `website-builder.models.widget`                                                         |
| Catalog, product detail, cart, checkout, order summary                 | `ecommerce.models.product`, `ecommerce.models.cart`, `ecommerce.models.order`, `ecommerce.models.widget`               |
| Article, category, tag, blog lists, article detail                     | `blog.models.article`, `blog.models.category`, `blog.models.tag`, `blog.models.widget`                                 |
| Login, registration, password reset, subject/account orchestration     | `rbac.models.identity`, `rbac.models.subject`, `rbac.models.widget`                                                    |
| Profile, author display, chat/thread/message display                   | `social.models.profile`, `social.models.chat`, `social.models.thread`, `social.models.message`, `social.models.widget` |
| Startup-specific overrides only                                        | `startup.models.widget` or `<owning-module>/<entity>/startup`                                                          |

## Manual Review Checklist

- Every retained runnable entry has a row in this matrix with `retained`.
- Every `migrated` or `replaced` row names a module/model/relation destination or
  explicitly states that the source has no Storybook surface.
- Admin shell, dashboards, model list/edit, drawer, relation management,
  editor, and UI primitives are classified as first-class rows.
- Startup is treated as overrides over singlepage contracts, not as a second
  copied app catalog.
